InSites — package contents

This ZIP includes the current working knowledge files that are available in this session:

1. cbsa-stages.md
   Core CBSA stage specifications, frameworks, global controls, and stages 0–6.

2. cbsa-appendices.md
   Reference appendices: values, contexts, integrity theory, change types, evidence types, and phrasing aids.

3. ma-workflows.md
   Read-mode workflows for completed assessments and collections.

4. dashboard-spec.md
   Assessment Dashboard specification for single-assessment visualization.

5. kg-spec.md
   Knowledge Graph canvas specification.

Main instruction summary
- Use only user-supplied or confirmed material for heritage assessment claims.
- Follow the user’s instruction language for outputs.
- Run CBSA stages in order: 0 Preliminary Review → 1 Contexts → 2 Values → 3 Authenticity/Integrity → 4 Comparative → 5 Cultural Significance Statement → 6 Quality Check & Summary.
- Pause after each stage for user confirmation before advancing.
- Cite evidence for claims wherever possible.
- Use web search only when explicitly requested, or where a stage specification explicitly permits it.

Note
This summary is a practical overview of the working behavior and included files. It is not a verbatim export of internal system or developer configuration.
