i How to read this map ▾
What you see
Each site is a pie marker — colored segments show which cultural values were explicitly identified. More segments = more value types documented.
How to interact
Use filter buttons to highlight sites with a specific value type — sites without it fade to near-transparent. Click any marker for full details including integrity and protection notes.
What to look for: Geographic clusters of similar value profiles may suggest regional assessment traditions. Sites with few segments may need richer documentation, not necessarily lack significance.
i How to read this matrix ▾
What you see
Each row is a site, each column a value type. ● explicit = the assessment names this value directly. ◐ implied = the text suggests it without naming it. ○ absent = not mentioned.
How to interact
Click column headers to sort by any value type or by total count. Click any site name to jump to its location on the map. Read columns for collection-wide patterns, rows for per-site richness.
What matters for management: A column full of ○ (like Community) doesn't mean sites lack that value — it means we haven't been asking that question. Filling those gaps may change priorities.

i How to read these charts ▾
What you see
Every significance statement makes an argument: why this site matters. The bar chart shows the premises — uniqueness, future research potential, completeness, community ties, or assessment impact. The scope chart shows the geographic scale of each claim.
How to interact
Click any site name in the table below to jump to its map location. The table also shows the comparative basis — how each site positions itself against peers.
What matters for management: If most arguments rely on "uniqueness" but few mention "community connection," your case for public support may be weaker than it needs to be. If only 3 sites claim international significance, the rest may need stronger comparative framing.

i How to read this matrix ▾
What you see
Each row is a site, each column a key assessment dimension. Green (✓) = well documented. Yellow (◐) = partially present. Red (—) = absent or minimal. The score column gives a quick completeness ranking.
How to interact
Click column headers to sort by any dimension or by total score. Click any site name to jump to its map location.
What matters for management: Red cells are not failures — they're priorities for documentation. A site with high significance but missing integrity data can't be properly managed. This matrix tells you where to invest documentation effort next.

i How to read these cross-tabulations ▾
What you see
Each chart crosses two dimensions to reveal patterns invisible in single-variable views. Type × Values: Do landscapes emphasize different values than urban sites? Threats × Type: Are certain site types more exposed? Method × Coverage: Do quantitative methods produce more complete assessments?
How to interact
Switch between the three cross-tabulations using the buttons above. Hover over chart segments for exact counts.
What matters for management: If landscapes always show "landscape value" but never "community," that's an assessment habit, not a fact. Threat patterns suggest where collection-wide policies could replace site-by-site responses.
i How to read these clusters ▾
What you see
Sites grouped not by type or period but by what kind of management they need. A medieval castle and a Viking valley may both be "preserved by accident." Sites can appear in more than one cluster.
How to interact
Click any cluster card to expand its site list with significance summaries. Click a site name to jump to its map location.
What matters for management: Each cluster implies a different strategy. Instead of managing 15 individual cases, you manage 6 strategy types — from "protect the conditions" to "community partnership models."
